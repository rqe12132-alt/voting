В ходе работы над дипломным проектом на тему «Разработка веб-приложения для проведения онлайн-голосований» были получены следующие результаты.

Выполнен анализ предметной области и определена актуальность разработки системы онлайн-голосований. Было установлено, что современные организации и общественные инициативы нуждаются в программном средстве, которое позволяет проводить прозрачные и управляемые голосования, обеспечивать достоверность результатов, защиту от повторного голосования и удобный доступ для широкого круга пользователей. Особое внимание было уделено проблемам: отсутствию централизованного учёта голосов, сложности контроля за повторным участием, недостатку инструментов для реального времени демонстрации результатов и отсутствию журнала аудита для проверки действий пользователей.

Проведён анализ существующих решений и подходов к организации онлайн-голосований. В ходе анализа были рассмотрены возможности современных платформ, их преимущества и ограничения. На основании проведённого исследования были сформулированы требования к разрабатываемому программному средству: поддержка регистрации и аутентификации пользователей с подтверждением электронной почты, разграничение прав доступа между администраторами и обычными пользователями, создание голосований с различными типами выбора (один вариант, несколько вариантов, да/нет), защита от повторного голосования, гибкие настройки видимости результатов, обновление результатов в реальном времени, журнал аудита, экспорт результатов в формат Excel и загрузка изображений для голосований и вариантов ответов.

Разработана архитектура программного средства. Серверная часть реализована на ASP.NET Core 8 с применением слоистой архитектуры и разделения ответственности. Presentation-слой представлен контроллерами API: `AuthController`, `PollsController`, `VotesController`, `AdminController`, `AuditController` и `UploadController`. Application-слой реализован сервисами бизнес-логики: `AuthService`, `PollService`, `VoteService`, `JwtService`, `EmailService`, `AuditService`, `ExcelExportService`. Infrastructure-слой включает репозитории для работы с базой данных MS SQL Server через Entity Framework Core и реализацию сервисов отправки почты и генерации токенов. Для описания схемы данных используется Fluent API в методе `OnModelCreating` класса `AppDbContext`, что позволяет централизованно управлять ограничениями, индексами и связями. Клиентская часть реализована как веб-приложение на HTML5, CSS3 и чистом JavaScript с использованием Bootstrap 5 и стандартного Fetch API. Для обмена данными в реальном времени применяется SignalR.

Реализован функционал авторизации и регистрации пользователей. В системе предусмотрена регистрация с указанием полного имени, email, пароля и идентификационного номера. Пароль хешируется с помощью алгоритма BCrypt. После регистрации генерируется шестизначный код подтверждения email, который отправляется пользователю через SMTP с использованием библиотеки MailKit. Код действителен в течение 24 часов. Также реализованы повторная отправка письма подтверждения и проверка статуса верификации при входе. При входе пользователя сервер проверяет email и пароль, после чего генерирует пару access token и refresh token. Access token используется для доступа к защищённым endpoint, а refresh token позволяет обновить сессию без повторного ввода пароля.

Реализована поддержка разграничения прав доступа. В системе выделяются обычные пользователи и администраторы. Первый зарегистрировавшийся пользователь автоматически получает права администратора. Последующие администраторы назначаются существующим администратором через панель управления с помощью секретного кода, передаваемого при регистрации. Проверка прав администратора выполняется на уровне контроллеров путём чтения claim `is_admin` из JWT-токена:

```csharp
private bool IsAdmin()
{
    return User.FindFirst("is_admin")?.Value?.ToLower() == "true";
}
```

Реализован модуль управления голосованиями. Администратор через панель управления может создавать голосования, указывать название, описание, тип (один вариант, несколько вариантов, да/нет), варианты ответов с изображениями, настройки видимости результатов, признак real-time обновления, а также дату и время начала и окончания. При создании голосование получает статус `Draft` (черновик), что позволяет внести правки до публикации. Для типа «Да / Нет» варианты создаются автоматически. Сервер проверяет корректность дат: дата начала и окончания не может быть в прошлом, а дата окончания должна быть позже даты начала. После публикации голосование переходит в статус `Active` и становится доступным для голосования. Администратор также может закрыть голосование досрочно, продлить срок окончания или удалить его. Все изменения статуса фиксируются в журнале аудита.

Реализован процесс голосования и подсчёт результатов. Пользователь выбирает один или несколько вариантов ответа в зависимости от типа голосования. Сервер получает идентификатор пользователя из JWT-токена, проверяет, что голосование активно и текущее время находится в пределах заданного срока, убеждается, что пользователь ещё не голосовал, и проверяет соответствие типа голосования количеству выбранных вариантов. Для защиты от повторного голосования в базе данных используется составной уникальный индекс по полям `UserId` и `PollId` в таблице `Votes`. После успешной проверки в базе данных создаются записи голосов. Расчёт результатов выполняется на основе агрегации голосов по вариантам ответа с вычислением процентного распределения:

```csharp
var counts = await _voteRepository.GetResultsAsync(pollId);
var total = counts.Values.Sum();

var results = poll.Options.Select(o =>
{
    var count = counts.ContainsKey(o.Id) ? counts[o.Id] : 0;
    var percentage = total > 0 ? Math.Round((double)count / total * 100, 2) : 0;
    return new VoteResultDto
    {
        OptionId = o.Id,
        OptionText = o.Text,
        Count = count,
        Percentage = percentage
    };
}).ToList();
```

В системе предусмотрены гибкие настройки видимости результатов: всегда видны, видны только после голосования или скрыты до окончания голосования. При запросе результатов сервер проверяет соответствующие правила и возвращает данные только в том случае, если пользователь имеет на это право.

Реализовано обновление результатов в реальном времени. Для обмена данными между сервером и клиентами в режиме реального времени используется SignalR. Клиентская часть устанавливает WebSocket-соединение с хабом `PollHub`, присоединяется к группе, соответствующей идентификатору голосования, и получает обновления результатов без необходимости ручного обновления страницы. При каждом новом голосе, если у голосования включён режим real-time, сервер отправляет актуальные результаты всем подключённым клиентам группы через событие `ResultsUpdated`.

Реализован журнал аудита. Система фиксирует важные действия пользователей: создание и удаление голосований, публикацию, закрытие, продление, голосование, вход в систему, загрузку файлов, назначение прав администратора и операции с тестовыми данными. Журнал аудита позволяет повысить прозрачность работы пользователей и упростить анализ изменений в системе. Администратор может просматривать журнал аудита в отдельной панели с пагинацией.

Реализован модуль экспорта результатов. Администратор может скачать результаты голосования в виде Excel-файла, содержащего название голосования, тип, статус, общее количество голосов, а также распределение голосов по вариантам с указанием количества и процента. Для формирования файла используется библиотека ClosedXML.

Разработан адаптивный пользовательский интерфейс. Frontend-приложение содержит страницы авторизации и регистрации, списка активных голосований, страницы голосования с real-time отображением результатов, истории голосований, профиля пользователя, подтверждения email, а также панели администратора для создания и управления голосованиями, просмотра журнала аудита и назначения администраторов. Интерфейс адаптирован для работы на разных разрешениях экрана, содержит формы ввода, таблицы, карточки, фильтры, модальные окна, уведомления и механизмы обработки ошибок.

Выполнено тестирование программного средства. Были разработаны тест-план, чек-листы и тест-кейсы. В ходе функционального тестирования проверялись сценарии регистрации, входа, подтверждения email, создания голосований, голосования, отображения результатов, real-time обновлений через SignalR, работы аудита и экспорта в Excel. Для тестирования API на этапе разработки использовался Swagger UI, который позволяет выполнять тестовые запросы к серверу. В ходе тестирования выявлены и исправлены дефекты, связанные с обработкой дат и отображением статусов голосований. После исправления дефектов проведена повторная проверка, подтвердившая корректность работы соответствующих функций.

Выполнено экономическое обоснование разработки программного средства. Были рассчитаны затраты на материалы, стоимость машинного времени, заработная плата, отчисления, прочие затраты, расходы на сопровождение и адаптацию программного обеспечения. На основании расчётов определена полная себестоимость разработки системы онлайн-голосований. Также рассмотрены возможные схемы использования программного продукта, включая его применение в образовательных учреждениях, общественных организациях и на корпоративных мероприятиях.

Практическая значимость разработанного программного средства заключается в том, что система онлайн-голосований может быть использована для организации прозрачных опросов, голосований и референдумов. Приложение обеспечивает достоверность результатов за счёт защиты от повторного голосования, фиксации всех действий в журнале аудита, использования хеширования паролей и верификации email. Наличие real-time обновлений и гибких настроек видимости результатов повышает удобство использования как для организаторов, так и для участников.

Новизна проекта заключается в разработке единого веб-приложения, объединяющего в одном программном продукте функции проведения голосований с различными типами выбора, защиты от повторного голосования на уровне базы данных, гибкого управления жизненным циклом голосования (черновик, активное, закрытое), real-time отображения результатов через SignalR, гибкой настройки видимости результатов, email-верификации пользователей, аудита действий, экспорта результатов в Excel, загрузки изображений и разграничения прав доступа на основе ролей и JWT-аутентификации. В отличие от простых опросников, разработанное программное средство обеспечивает полный цикл управления голосованием: от создания и публикации до голосования, подсчёта результатов и аудита.
