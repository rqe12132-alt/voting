# 🗳️ VotingApp — Полная инструкция по запуску и демонстрации

> **Стек:** ASP.NET Core 8 + EF Core + MS SQL Server + SignalR + HTML/CSS/JS + Bootstrap 5

---

## 📋 Требования

- **Fedora Linux** (или любой Linux с Podman)
- **.NET 8 SDK** (уже установлен в `~/.dotnet`)
- **Podman** (или Docker)
- **Python 3** (для frontend-сервера)
- Современный браузер (Firefox, Chrome)

---

## 🚀 БЫСТРЫЙ ЗАПУСК (3 команды)

### Шаг 1. Перейти в папку проекта
```bash
cd ~/code/diplom
```

### Шаг 2. Запустить все сервисы
```bash
./start.sh
```
Этот скрипт автоматически:
1. Проверит и запустит MSSQL Server в Podman
2. Запустит Backend API на `http://localhost:5000`
3. Запустит Frontend на `http://localhost:8080`

### Шаг 3. Создать демо-данные (опционально, для защиты диплома)
```bash
./demo-data.sh
```
Создаст 3 голосования и 50+ тестовых пользователей с голосами.

## 🔧 РУЧНОЙ ЗАПУСК (если скрипт не сработал)

### 1. Запуск MSSQL Server (Podman)

Проверьте, запущен ли контейнер:
```bash
podman ps
```

Если **не запущен**:
```bash
podman start sqlserver
```

Если контейнера **нет** (первый запуск):
```bash
podman run -e "ACCEPT_EULA=Y" \
  -e "MSSQL_SA_PASSWORD=YourStrong@Pass1" \
  -p 1433:1433 --name sqlserver \
  -d mcr.microsoft.com/mssql/server:2022-latest
```

Подождите **15 секунд** пока БД инициализируется.

### 2. Применение миграций (первый запуск)

```bash
cd ~/code/diplom/voting-backend/VotingApp
export DOTNET_ROOT="$HOME/.dotnet"
export PATH="$DOTNET_ROOT:$PATH"
dotnet ef database update
```

### 3. Запуск Backend

```bash
cd ~/code/diplom/voting-backend/VotingApp/bin/Debug/net8.0
export DOTNET_ROOT="$HOME/.dotnet"
export PATH="$DOTNET_ROOT:$PATH"
dotnet VotingApp.dll --urls "http://localhost:5000"
```

> **Важно:** Оставьте эту вкладку терминала открытой! Backend должен работать постоянно.

### 4. Запуск Frontend (в новой вкладке терминала)

```bash
cd ~/code/diplom/voting-frontend
python3 -m http.server 8080
```

> Оставьте эту вкладку тоже открытой.

## 🎓 КАК ПОКАЗАТЬ РАБОТОСПОСОБНОСТЬ (пошаговая демонстрация)

### Сценарий 1: Регистрация и создание голосования (Админ)

**Откройте:** `http://localhost:8080/login.html`

1. **Переключитесь на вкладку «Регистрация»**
2. **Введите данные:**
   - ФИО: `Администратор`
   - Email: `admin@demo.ru`
   - Пароль: `password123`
   
   *Первый зарегистрировавшийся пользователь станет администратором автоматически.*
3. **Нажмите «Зарегистрироваться»**
   - Система автоматически перенаправит на главную страницу
4. **Нажмите «⚙️ Админка»** в шапке
5. **Вкладка «➕ Создать голосование»**
   - Название: `Лучший язык программирования`
   - Описание: `Какой язык вы предпочитаете?`
   - Тип: `Один вариант`
   - Варианты: `C#`, `Python`, `JavaScript`
   - Видимость результатов: `Всегда видны`
   - Real-time: ✅ (галочка включена)
   - Нажмите **«💾 Сохранить черновик»**
6. **Перейдите на вкладку «📋 Управление»**
   - Найдите созданное голосование
   - Нажмите **«🚀 Опубликовать»**
   - Статус сменится на **Active**

### Сценарий 2: Голосование и Real-time результаты

**В новой вкладке браузера** (без выхода из админки):

1. Откройте `http://localhost:8080/login.html`
2. **Зарегистрируйте обычного пользователя:**
   - Email: `user@demo.ru`
   - Пароль: `password123`
3. **На главной странице** (`index.html`) появится опубликованное голосование
4. **Нажмите «Проголосовать»**
5. **Выберите вариант** (например, `C#`) и нажмите **«🗳️ Проголосовать»**
6. **Вы увидите результаты** с:
   - Количеством голосов
   - Процентами
   - Прогресс-барами
7. **Real-time демонстрация:**
   - **Откройте ещё одну вкладку** с тем же голосованием
   - Проголосуйте во второй вкладке другим вариантом
   - **Наблюдайте**, как в первой вкладке результаты обновились **автоматически** без перезагрузки (через SignalR)

### Сценарий 3: Защита от повторного голосования

1. Попробуйте нажать «Проголосовать» ещё раз
2. **Ожидаемый результат:** Система отклонит попытку (пользователь уже голосовал)

### Сценарий 4: Разные типы голосований

**Через админку** создайте:
1. **«Да / Нет» голосование:**
   - Тип: `Да / Нет`
   - Варианты добавятся автоматически (Да / Нет)
2. **«Несколько вариантов» голосование:**
   - Тип: `Несколько вариантов`
   - Варианты: `Frontend`, `Backend`, `DevOps`, `QA`
   - Пользователь сможет выбрать **несколько** чекбоксов

### Сценарий 5: Настраиваемая видимость результатов

1. Создайте голосование с **«Скрыты до окончания»**
2. Опубликуйте его
3. Проголосуйте обычным пользователем
4. **Ожидаемый результат:** Результаты не отображаются (403)
5. **В админке удалите** это голосование (или создайте новое с другой видимостью)

## ❗ Возможные проблемы

| Проблема | Решение |
|----------|---------|
| `podman: command not found` | Установите Podman: `sudo dnf install podman` |
| `dotnet: command not found` | Выполните: `export DOTNET_ROOT="$HOME/.dotnet" && export PATH="$DOTNET_ROOT:$PATH"` |
| `Failed to bind to address http://localhost:5000` | Порт занят. Остановите процесс: `pkill -f VotingApp` |
| `Login failed: 401` | Проверьте email/пароль. Или токен истёк — обновите страницу |
| Страница белая/не загружается | Проверьте, запущены ли все 3 сервиса (`podman ps`, `curl localhost:5000`, `curl localhost:8080`) |
| SignalR не работает | Проверьте, что в `Program.cs` не закомментирован `app.MapHub` |
